# IPv4 和 IPv6 的区别

- **IPv4**：老一代 IP，格式像 `1.2.3.4`
- **IPv6**：新一代 IP，格式像 `2408:xxxx:xxxx::1`
- **本质区别**：IPv6 地址数量更多，设计更现代；IPv4 更通用，兼容性最好

---

## 访问 Claude / ChatGPT 时，网站读到的是哪个 IP？

取决于你访问时**实际走的是哪条网络栈**：

- 如果你的 VPS 和目标网站**都支持 IPv6**，且系统优先走 IPv6，网站看到的就是 **你的 IPv6**
- 否则通常看到的是 **你的 IPv4**

所以对 ChatGPT / Claude / Gemini 来说：

- 可能读到 IPv4
- 也可能读到 IPv6
- **不是固定的**

最常见情况：

- 浏览器 / 系统优先 IPv6 → 网站看到 IPv6
- 没有走 IPv6 或不通 → 网站看到 IPv4

一句话：**网站只能看到"你这次连接实际使用的那个出口 IP"，不是同时看到两个。**

---

## 为什么这对防封很重要？

如果你开了代理但没开 TUN 模式，可能出现这种情况：

- HTTP 流量走代理 → 网站看到 **VPS 的 IPv4**（干净）
- IPv6 流量没有被代理接管 → 直连 → 网站看到 **你本机真实的 IPv6**

两个 IP 同时出现，就会触发风控。

**解决方案**：开启 TUN 模式，让所有流量都经过 Clash，包括 IPv6。
